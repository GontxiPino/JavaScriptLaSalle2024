Apuntes de Variables en JavaScript

Definición

En JavaScript, las variables se utilizan para almacenar datos. Se pueden declarar utilizando var, let y const.

Declaración de Variables

Las variables se pueden declarar de las siguientes maneras:

  • var: Declara una variable con alcance global o de función. 
    var nombre = "Juan";
  • let: Declara una variable con alcance de bloque. 
    let edad = 30;
  • const: Declara una constante con alcance de bloque. Su valor no puede cambiar. 
    const PI = 3.1416;

Alcance de Variables

El alcance determina dónde se puede acceder a una variable:

  • Global: Una variable declarada fuera de cualquier función tiene un alcance global y puede ser accedida desde cualquier parte del código. 
    var globalVar = "Soy global";
    console.log(globalVar); // "Soy global"
  • De función: Una variable declarada dentro de una función solo puede ser accedida desde dentro de esa función. 
    function miFuncion() {
        var funcionVar = "Soy local de función";
        console.log(funcionVar); // "Soy local de función"
    }
    miFuncion();
    // console.log(funcionVar); // Error: funcionVar is not defined
  • De bloque: Una variable declarada con let o const dentro de un bloque solo puede ser accedida desde dentro de ese bloque. 
    if (true) {
        let bloqueVar = "Soy local de bloque";
        console.log(bloqueVar); // "Soy local de bloque"
    }
    // console.log(bloqueVar); // Error: bloqueVar is not defined

Hoisting

El hoisting es un comportamiento en JavaScript en el que las declaraciones de variables y funciones se mueven al inicio de su contexto de ejecución.

  • Las variables declaradas con var se inicializan como undefined al principio del contexto de ejecución. 
    console.log(hoistedVar); // undefined
    var hoistedVar = "Soy hoisted";
  • Las variables declaradas con let y const no se inicializan hasta que su declaración es evaluada en tiempo de ejecución. 
    // console.log(noHoistedLet); // Error: noHoistedLet is not defined
    let noHoistedLet = "No soy hoisted";

Buenas Prácticas

  • Utiliza let y const en lugar de var para evitar errores relacionados con el alcance y el hoisting.
  • Declara todas las variables al principio de su contexto de ejecución.
  • Utiliza nombres de variables descriptivos y en camelCase. 
    let nombreUsuario = "Juan";
    const MAX_INTENTOS = 3;

Ejemplo Completo

function ejemploVariables() {
    var globalVar = "Soy global";
    let bloqueVar = "Soy local de bloque";
    const CONSTANTE = "No cambio";

    if (true) {
        let otraBloqueVar = "Otra local de bloque";
        console.log(otraBloqueVar); // "Otra local de bloque"
    }

    console.log(globalVar); // "Soy global"
    console.log(bloqueVar); // "Soy local de bloque"
    console.log(CONSTANTE); // "No cambio"

    // console.log(otraBloqueVar); // Error: otraBloqueVar is not defined
}
ejemploVariables();
